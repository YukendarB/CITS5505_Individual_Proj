AI Reflection Log

This page documents how AI tools were used during the development of DevQuest and critically evaluates their usefulness, limitations, risks and ethical implications in a coding project.

Overview

How AI was used in this project

Large language models were used throughout the DevQuest project to support planning, interface design, debugging and feature implementation. AI was especially helpful in generating starter structures for HTML pages, suggesting CSS layout improvements and helping reason through the logic required for the quiz page, which was the most technically demanding part of the assignment.

However, AI outputs were not accepted blindly. In many cases, the generated solutions needed revision because they were either too generic, did not fully fit the project constraints or failed to account for browser-specific issues that appeared during testing. This meant AI was most useful when treated as a drafting and support tool rather than a source of final answers.

Examples

Selected examples of AI use

Example 1 - Initial page structure

Prompt used:

Give me a clean base HTML structure for the required pages in a static web project with shared navigation and styling.

What the AI produced:

AI generated starter HTML page structures and suggested a shared layout approach for navigation, containers and reusable styling.

What I changed:

I adapted the structure to fit my actual project theme, file names and page content. I also refined the layout so the site felt more coherent and aligned with the DevQuest identity.

Limitation noticed:

The generated structure was helpful as a draft, but it was generic. It still required design decisions and editing to make it suitable for the assignment.

Example 2 - Loading quiz questions from JSON

Prompt used:

How can I load quiz questions from a local JSON file and render them dynamically in JavaScript for my quiz page?

What the AI produced:

AI suggested using fetch() to load the JSON file and then dynamically generate the quiz questions in the DOM.

What I changed:

I adapted the generated logic to fit my quiz structure, question randomisation, validation flow and scoring system. I also integrated it with my existing JavaScript file and page structure rather than copying the solution directly.

Limitation noticed:

The original suggestion did not fully account for the browser error I faced when opening the project locally. Because the quiz questions were stored in a JSON file, the browser did not always allow fetch() to load them correctly from the local file system. This required debugging, understanding why the request failed and testing the project in a proper local development environment.

Example 3 - Choosing a color palette

Prompt used:

What factors should I consider when choosing a dark theme color palette for a web project to ensure it looks polished and remains easy to read?

What the AI produced:

AI suggested considering contrast, text readability, accent color restraint, consistent surface colors and how buttons, cards and highlights would appear across the full interface.

What I changed:

I used those factors to review the DevQuest theme rather than treating the response as a finished palette. I kept the dark space-inspired background, then adjusted accent colors and card contrast so the tutorial, quiz, reflection, and CV pages felt consistent and stayed readable.

Limitation noticed:

The advice was useful as a checklist, but it could not show whether the colors would actually feel balanced in my layout. I still needed to test the palette on the real pages because some accents that seemed appropriate in theory became too strong when repeated across cards, buttons and headings.

Example 4 - Designing the favicon

Prompt used:

Create favicon ideas for DevQuest. The site is about learning web development, with a coding and rocket theme. I want the favicon to feel simple, memorable and relevant to my personal project branding.

What the AI produced:

AI suggested several icon directions combining coding symbols, rocket imagery and minimal branding concepts.

What I changed:

I refined the idea based on the project identity and selected a simpler direction that would remain recognisable at very small sizes. I focused on clarity and branding rather than detail.

Limitation noticed:

AI-generated design ideas were helpful for exploration, but they still required personal judgment to choose something visually clear original and suitable as a favicon.

Critical Evaluation

Reliability, risks and limitations

AI was useful for quickly creating first drafts, but the work still depended on testing and judgement. The clearest technical example was the quiz page, where loading questions from a local JSON file caused browser-specific errors when the project was opened directly from the file system.

This showed that AI-generated code can look correct while still failing in the actual browser environment. I had to debug the failed JSON request, understand why fetch() behaves differently with local files and test the quiz in a proper local development environment instead of assuming the first solution was complete.

AI design suggestions also needed practical testing. Some color palette and favicon ideas sounded suitable in text, but they had to be adjusted once applied to the real DevQuest interface so the pages stayed readable, consistent, and not overly distracting.

Ethical Considerations

Responsible use of AI

The use of AI in this project raises important questions about authorship, understanding and academic integrity. In this project, AI was used as a support tool rather than a substitute for learning. All generated code and writing were reviewed, edited and tested before being incorporated into the project.

This process also made it clear that relying on AI without understanding the output would be risky. In several cases, such as browser crashes, reward API failures and quiz filtering issues, a superficial use of AI would have produced a broken submission. Ethical use therefore required active verification, personal judgement and responsibility for the final result.

Conclusion

Final Reflection

Overall, AI significantly supported the development of DevQuest by accelerating ideation, drafting content and suggesting possible technical solutions. At the same time, the project demonstrated that AI is not a replacement for reasoning, debugging or understanding assignment requirements. Its outputs were most useful when treated as a starting point that needed to be tested, questioned and refined.

The experience showed that AI can enhance software development productivity, but only when combined with critical thinking and responsible use. In this project, the final quality of the work depended not on AI alone, but on how effectively its suggestions were evaluated and improved.
