MODULE 01

HTML: Structure and Meaning

HTML is the foundation of a webpage. It tells the browser what each piece of content is, such as a heading, paragraph, link, image, navigation area, article, form control, or table row.

Document foundation

Use <!DOCTYPE html>, <html>, <head>, and <body> to give the page a reliable starting structure.

Content elements

Use <h1>, paragraphs, <a> links, images with alt, and <em> for meaningful emphasis.

Semantic regions

Use <header>, <nav>, <main>, <section>, <article>, and <footer> to describe layout purpose.

Lists and tables

Lists group related points, while tables should be saved for row-and-column data such as schedules, scores, or comparisons.

Forms and labels

Forms collect user input. Labels connect text to controls, and input types such as radio guide how users answer.

Head vs body

The <head> stores metadata and links to resources; the <body> contains what users see and interact with.

Learning focus

  • Start with one clear page structure before adding styling.
  • Choose elements based on meaning, not just how they look.
  • Use labels and alt text so the page works for more people and tools.
Example: semantic HTML page structure
<!DOCTYPE html>
<html lang="en">
<head>
  <!-- The title appears in the browser tab, not in the visible page body. -->
  <title>DevQuest Lesson</title>
</head>
<body>
  <header>
    <h1>HTML Foundations</h1>
    <nav aria-label="Lesson navigation">
      <a href="#css-module">Next: CSS</a>
    </nav>
  </header>

  <main>
    <article>
      <h2>Why structure matters</h2>
      <p>Use <em>semantic elements</em> to make meaning clear.</p>
      <img src="images/devquest-logo.svg" alt="DevQuest logo">
    </article>

    <label for="topicChoice">Choose one topic</label>
    <input id="topicChoice" type="radio" name="topic">
  </main>
</body>
</html>
Example: form and table content
<!-- Labels help users and assistive technology understand each control. -->
<form>
  <fieldset>
    <legend>Choose one quiz mode</legend>

    <label for="modeHtml">
      <input id="modeHtml" type="radio" name="mode" value="html">
      HTML practice
    </label>
  </fieldset>
</form>

<!-- Use tables for real tabular data, not for page layout. -->
<table>
  <tr>
    <th>Module</th>
    <th>Focus</th>
  </tr>
  <tr>
    <td>HTML</td>
    <td>Structure and meaning</td>
  </tr>
</table>

Why it matters

Semantic HTML is not just cleaner code. It helps screen readers, search tools, teammates, and future you understand the page. When the structure is clear, styling and JavaScript can build on it without fragile guesswork.

MODULE 02

CSS: Styling and Layout

CSS controls how the HTML looks. It handles color, typography, spacing, layout, hover states, responsive rules, and reusable visual patterns.

Selectors and reuse

#intro targets one id, .card targets reusable classes, and shared classes keep styles consistent.

Box model and spacing

padding is inside the border, margin is outside it, and border-radius rounds corners.

Layout and responsiveness

flex handles one-dimensional layouts, Grid handles rows and columns, and @media adapts to viewport size.

Pseudo-classes

:hover lets the interface respond visually when a pointer is over an element, which can make controls feel clearer.

Typography and color

Readable font sizes, contrast, and consistent colors help users scan a page and understand what matters first.

Practical layering

Positioned elements can overlap. z-index controls which layer appears above another when stacking matters.

Learning focus

  • Use class selectors for design patterns that appear more than once.
  • Think in boxes: content, padding, border, and margin all affect spacing.
  • Keep layout rules responsive so the page remains readable on smaller screens.
Example: reusable CSS card pattern
/* A reusable class keeps repeated content visually consistent. */
.lesson-card {
  background: rgba(255, 255, 255, 0.05);
  border: 1px solid rgba(255, 255, 255, 0.12);
  border-radius: 0.9rem;
  padding: 1rem;
  transition: border-color 0.25s ease, transform 0.25s ease;
}

/* Hover feedback helps users notice interactive or important areas. */
.lesson-card:hover {
  border-color: rgba(99, 102, 241, 0.52);
  transform: translateY(-2px);
}

/* Grid creates a two-dimensional layout for multiple cards. */
.lesson-grid {
  display: grid;
  gap: 1rem;
  grid-template-columns: repeat(3, 1fr);
}

@media (max-width: 768px) {
  .lesson-grid {
    grid-template-columns: 1fr;
  }
}
Example: selector types and readable text
/* Element selector: affects every paragraph. */
p {
  line-height: 1.6;
}

/* Class selector: reusable across many cards. */
.notice-card {
  color: #e2e8f0;
  background: rgba(15, 23, 42, 0.85);
}

/* ID selector: use sparingly for one unique element. */
#lessonSummary {
  border-left: 4px solid #6366f1;
}

/* Pseudo-class: provide feedback when the card is hovered. */
.notice-card:hover {
  border-color: #ec4899;
}

Why it matters

Separating content from style keeps projects easier to change. HTML can focus on meaning, while CSS controls presentation, spacing, and responsiveness from one place instead of scattering design decisions through the page.

MODULE 03

JavaScript: Behavior and Feedback

JavaScript lets the page react after it loads. It can listen for clicks, change DOM content, validate forms, store small values, and fetch data from another file or public API.

Events

DOMContentLoaded waits for the document, click reacts to buttons, and preventDefault() can stop normal form submission.

DOM updates

querySelector() finds an element, addEventListener() attaches behavior, and text or classes can change instantly.

Data and logic

let can be reassigned, const should not be, === checks value and type, JSON stores data, and fetch() returns a Promise.

Form validation

JavaScript can check whether a user has answered required fields before allowing a form to submit.

Local storage

localStorage stores small values in the browser, which is useful for attempt history or user preferences.

AJAX requests

fetch() can load JSON from another file or public API without refreshing the whole page.

Learning focus

  • Select the elements you need before trying to change them.
  • Use events to respond to user actions at the right moment.
  • Update the DOM carefully so the interface gives useful feedback without becoming noisy.
Example: DOM update from a button click
// Wait until the HTML exists before selecting page elements.
document.addEventListener("DOMContentLoaded", () => {
  const button = document.querySelector("#demoButton");
  const output = document.querySelector("#demoOutput");

  // Use let because the counter changes after every click.
  let clickCount = 0;

  button.addEventListener("click", () => {
    clickCount += 1;
    output.textContent = `DOM update complete. Click count: ${clickCount}`;
    output.classList.add("demo-active");
  });
});
Example: quiz-style validation and JSON loading
// Stop a form from submitting until the answer is complete.
form.addEventListener("submit", (event) => {
  const selectedAnswer = form.querySelector("input[name='question']:checked");

  if (!selectedAnswer) {
    event.preventDefault();
    message.textContent = "Choose an answer before submitting.";
  }
});

// Load structured question data without hard-coding it into the page.
fetch("data/questions.json")
  .then((response) => response.json())
  .then((questions) => {
    localStorage.setItem("lastQuestionCount", String(questions.length));
  });

Why it matters

Interactivity improves user experience because people get feedback at the moment they act. A quiz can show validation, a button can reveal extra information, and a page can load JSON without forcing a full reload.
